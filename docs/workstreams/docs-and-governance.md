# Docs And Governance Workstream

Branch: `workstream/docs-and-governance`

## Purpose

Refine the contribution, safety, review, security, and maintainer practices for the founding repository.

## Scope

- Improve contribution guide and review expectations.
- Refine issue templates and pull request template.
- Improve security policy for future audio/video handling.
- Document maintainer workflow and branch hygiene.

## Guardrails

- Do not add private contact details unless the maintainer explicitly provides a public reporting address.
- Do not weaken privacy, security, or review expectations.
- Do not add paid infrastructure, secrets, deployments, or analytics.
