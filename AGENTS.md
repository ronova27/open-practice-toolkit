# Agent Guide

This repository is an early open-source bootstrap for OpenPractice Toolkit.

## Operating Rules

- Do practical work directly when tools and files are available.
- Keep changes small, reviewable, and scoped to the requested workstream.
- Do not add secrets, API keys, tokens, or private personal data.
- Do not configure paid infrastructure, billing, deployment, or hosted services.
- Do not overpromise audio transcription, video analysis, or AI behavior.
- Prefer boring, maintainable structure over impressive complexity.
- Preserve local-first and privacy-respecting defaults.

## Branch Discipline

- Work only on the branch named in the task.
- Avoid touching unrelated modules.
- Use small commits or small patch sets.
- Add tests, research notes, or verification steps when practical.
- Stop before destructive actions, billing changes, deployments, or public submissions.

## Project Boundaries

Core practice features should remain usable without a paid API. Optional AI-assisted workflows may be explored later, but they must be clearly separated from required local practice features.

## Protocol

Record meaningful bootstrap or maintenance turns in `PROTOCOL.md` when the work changes repository structure, branch strategy, governance, or public application material.
